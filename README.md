# Windows File Server Security & DFS Standardizer

Профессиональный PowerShell инструмент для автоматизированного приведения структуры каталогов файлового сервера к строгому стандарту безопасности на основе ролевой модели **AGDLP** (Accounts -> Global -> Domain Local -> Permissions) с последующей публикацией ресурсов в инфраструктуре распределенной файловой системы (DFS Namespace).



## 🌟 Возможности инструмента

* **Автоматическое разрешение зависимостей**: При первом запуске скрипт проверяет наличие необходимых RSAT-модулей командной строки (`ActiveDirectory` и `DFSN`) и производит их бесшовную установку через Server Manager.
* **Изоляция прав доступа**: Полное отключение унаследованных ACE (Access Control Entries) с верхних уровней диска без копирования старых дескрипторов (полная очистка от скрытых прямых назначений разрешений).
* **Генерация ролевых групп**: Автоматическое создание доменных локальных групп безопасности (для сценариев Чтения и Изменения данных) в целевом контейнере Active Directory.
* **Управление общими папками**: Развертывание скрытых (или явных) сетевых ресурсов SMB с обязательным включением механизма **Access-Based Enumeration (ABE)** (пользователи видят только те папки, к которым имеют доступ).
* **Публикация в DFS**: Автоматическое связывание созданных сетевых шар со сквозным доменным пространством имен DFS без нарушения текущих существующих путей.
* **Безопасный прогон (-WhatIf)**: Полная интеграция с нативным механизмом симуляции изменений Windows PowerShell.
* **Глубокий аудит**: Режим выгрузки детальных CSV-отчетов по текущему состоянию NTFS-прав и сетевых ресурсов перед началом работ.

## 🛠️ Архитектура обработки каталога

Для каждой обрабатываемой папки инструмент последовательно выполняет следующие операции:

1.  Формирует имя базовой группы (убирает спецсимволы, заменяет пробелы на знаки подчеркивания).
2.  Создает в Active Directory две группы: `<имя_папки>_FOLDER_FULL` и `<имя_папки>_FOLDER_READ-ONLY`.
3.  Очищает ACL папки и собирает чистый эталонный список доступа: `SYSTEM (FC)`, `Administrators (FC)`, `Domain Admins (FC)`, `CREATOR OWNER (FC)`, а также добавляет созданные доменные группы со строго разграниченными правами (`Modify` или `ReadAndExecute`).
4.  Создает скрытую сетевую шару `<имя_папки>$` и назначает Share-разрешения только для Администраторов и созданных ролевых групп.
5.  Добавляет линк в DFS Namespace: `\\domain\namespace\<имя_папки>` -> `\\server\<имя_папки>$`.

## 📦 Быстрый старт и конфигурация

Все параметры работы вынесены в изолированный файл конфигурации `config.json`. Перед использованием утилиты отредактируйте параметры под архитектуру вашей организации:

```json
{
    "Mode": ["Audit"],
    "Root": "D:\\",
    "DomainDNS": "company.local",
    "DomainNB": "COMPANY",
    "ServerHost": "FS01",
    "GroupOU": "OU=Resource Groups,DC=company,DC=local",
    "DfsNamespace": "\\\\company.local\\PublicNamespace",
    "ReportDir": "C:\\Temp\\FS-SecurityAudit",
    "FullSuffix": "_FOLDER_FULL",
    "RoSuffix": "_FOLDER_READ-ONLY",
    "SkipFolders": ["SHARED_FOLDERS", "SYSTEM_RESERVED"],
    "TargetFolders": [],
    "RWRight": "Modify",
    "HiddenShares": true
}
```

Описание критических параметров:
Root: Абсолютный локальный путь к диску, каталоги которого необходимо стандартизировать.

SkipFolders: Список папок верхнего уровня, которые скрипт обязан полностью игнорировать при пакетной работе (например, служебные или публичные папки).

TargetFolders: Ограничение списка обработки конкретными папками (идеально для пилотных проектов). Если массив пуст [] — обрабатываются все дочерние папки корневого диска.

🚀 Примеры использования
Скрипт должен запускаться из консоли PowerShell, открытой от имени Администратора, под учетной записью, имеющей права на управление целевой OU в Active Directory, локальным сервером и DFS-пространством.

1. Режим сбора данных и генерации отчетов (Аудит)
Анализирует текущую структуру, не вносит изменения. Результаты (CSV и таблицы) сохраняются в каталог ReportDir.

PowerShell
.\Set-FolderSecurity.ps1 -Mode Audit
2. Безопасная симуляция изменений (Пилотный запуск)
Моделирование полного цикла развертывания для конкретных папок с выводом предполагаемых изменений на экран (благодаря -WhatIf реальные данные на сервере и в AD не затрагиваются):

PowerShell
.\Set-FolderSecurity.ps1 -Mode Provision -TargetFolders "Baku", "Ganja" -WhatIf
3. Боевое применение на выбранных каталогах
Реальное создание групп в AD, сброс NTFS, поднятие шар и DFS для указанного списка:

PowerShell
.\Set-FolderSecurity.ps1 -Mode Provision -TargetFolders "Baku", "Ganja"
4. Автоматизация всей файловой системы (Ковровая обработка)
Применение стандарта ко всем подкаталогам диска, за исключением папок, находящихся в массиве SkipFolders:

PowerShell
.\Set-FolderSecurity.ps1 -Mode Provision
5. Точечный запуск отдельного этапа (Например, только DFS)
Если вам необходимо выполнить только интеграцию в DFS-пространство (например, если группы и шары уже созданы):

PowerShell
.\Set-FolderSecurity.ps1 -Mode SetDFS
🔒 Требования к среде выполнения
Операционная система: Windows Server 2012 R2 / 2016 / 2019 / 2022 / 2025.

Версия оболочки: PowerShell 5.1 и выше.

Разрешения контекста: Членство в группах Domain Admins (или делегированные права на создание объектов Group в рамках целевой OU) и Local Administrators на файловом сервере.
